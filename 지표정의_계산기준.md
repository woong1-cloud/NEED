# 커머스 대시보드 PRD (개발 전달용)

## 4. 화면 요구사항

### 4.1 공통
- 탭 구성: `요약판` / `상세판`
- 비교 기준 토글: `전년` / `전주` (기본값 `전년`)
- 표시 단위:
  - 금액: 원 또는 만원(화면 정책에 맞춰 고정)
  - 비율: `%`
- 분모 0 또는 데이터 없음인 경우: `-` 표기

### 4.2 요약판
- 상단 핵심 카드: 매출, UV, 전환율, 반품율
- 성과 하이라이트 블록
- 목표 달성률 비교 블록
- 하단 `상품 판매 Top 20` 테이블

### 4.3 상세판
- 상세판 상단 헤더
  - 제목/기준일자
  - 조회 모드 버튼: `월 누적`, `주간`, `일별 조회`
  - 비교 기준 버튼: `전년`, `전주`
  - 일별 모드에서만 스냅샷 날짜 선택 노출
- 헤더 아래 핵심지표 표(표 형태)
  - 컬럼: `핵심지표 | 실적 | 목표 | 달성률 | (필터값) | 성장률`
- 지표 섹션 순서
  1. 매출 / UV
  2. 핵심 지표(유입/전환/객단가)
  3. 유입 상세(신규/재방문/앱유입/MAU)
  4. 전환 상세(퍼널)
  5. 구매 후 현황(반품율/재구매율)
  6. 서비스 현황(앱 다운로드/신규 가입자/쿠폰)
  7. 상품 대시보드
     - 효율좋은 페이지 Top 5 (매출 기준)
     - 효율좋은 카테고리 Top 5 (매출 기준, 경로 기준)
  8. 상품 판매 Top 20

### 4.4 상품 대시보드 요구사항
- 위치: 상세판에서 `상품 판매 Top20` 바로 위
- 블록 1: 효율좋은 페이지 Top 5
  - 컬럼: 순위, 페이지(URL), UV, 매출, 매출/UV
- 블록 2: 효율좋은 카테고리 Top 5
  - 컬럼: 순위, 카테고리 경로, 주문, 매출, 매출/주문
- 정렬 기본: 매출 내림차순

## 5. 지표 정의 및 계산 기준

## 5.1 공통 기준
- 시간대: KST
- 비교 기준 기본값: `전년(year)`
- 비교 선택값: `전년(year)` / `전주(week)`
- 비교 기준 전환 시, 비교값/성장률/비교 문구가 동일 기준으로 함께 변경되어야 함

### 5.2 핵심 지표

#### 매출 (Revenue)
- 정의: 결제완료 주문의 매출 합계
- 계산식: `매출 = Σ(order_amount)`

#### UV (Unique Visitors)
- 정의: 기간 내 유니크 방문자 수
- 계산식: `UV = COUNT(DISTINCT visitor_key)`
- visitor_key 정책: `user_id` 우선, 없으면 `device_id + cookie`

#### 전환율 (CVR)
- 정의: 방문 대비 구매 전환 비율
- 계산식: `전환율(%) = (주문건수 / UV) * 100`

#### 객단가 (AOV)
- 정의: 주문 1건당 평균 매출
- 계산식: `객단가 = 매출 / 주문건수`

#### 반품율 (Return Rate)
- 정의: 주문 대비 반품 비율 (금액 기준 권장)
- 계산식: `반품율(%) = (반품금액 / 매출) * 100`

#### 재구매율 (Repurchase Rate)
- 정의: 구매 고객 중 2회 이상 구매 고객 비율
- 계산식: `재구매율(%) = (2회 이상 구매 고객 수 / 구매 고객 수) * 100`

### 5.3 퍼널 지표
- 단계: 방문 → 상품조회 → 장바구니 → 결제완료
- 계산식
  - `상품조회율 = 상품조회수 / 방문수 * 100`
  - `장바구니율 = 장바구니수 / 상품조회수 * 100`
  - `결제완료율 = 결제완료수 / 방문수 * 100`

### 5.4 서비스 지표
- 앱 다운로드: 기간 내 앱 설치 이벤트 수
- 신규 가입자: 기간 내 신규 회원 수
- 쿠폰 사용 금액: 쿠폰 할인 총액
- 쿠폰 건당 평균 할인액: `쿠폰 사용 금액 / 쿠폰 사용 건수`

### 5.5 상단 표(핵심지표 표) 계산
- 달성률: `달성률(%) = (실적 / 목표) * 100`
- 필터값(비교값): 비교 기준(`전년` 또는 `전주`) 실적
- 성장률: `성장률(%) = (실적 - 비교값) / 비교값 * 100`

### 5.6 상품 대시보드 계산
- 효율좋은 페이지 Top5: `매출/UV = 페이지 매출 / 페이지 UV`
- 효율좋은 카테고리 Top5: `매출/주문 = 카테고리 매출 / 카테고리 주문`

### 5.7 예외 처리
- 분모가 0인 비율값: `-`
- 비교값이 0 또는 null인 성장률: `-`
- 비율 기본 자리수: 소수점 1자리(필요 시 2자리까지)

## 6. 기간/비교 동작 규칙

### 6.1 월 누적
- 기간: 기준월 1일 00:00:00 ~ 기준일
- 비교
  - 전년: 전년 동일 기간
  - 전주: 직전 비교구간

### 6.2 주간
- 기준일(anchorDate) 기준 **직전 주 월~일(7일)**
- 예시
  - 기준일 `2025-04-27(일)` → 주간 구간 `2025-04-14(월) ~ 2025-04-20(일)`
- 비교
  - 전년: 전년 동주
  - 전주: 직전 주 동구간

### 6.3 일별(스냅샷)
- 사용자가 선택한 `YYYY-MM-DD` 1일치 데이터
- 선택한 날짜는 상세판 헤더 하단에 표시
- 비교
  - 전년: 전년 동요일
  - 전주: 전주 동요일(정책 고정)

### 6.4 동작 연동 규칙
- 조회 모드 변경 시: 모든 상세 섹션 데이터 일괄 갱신
- 비교 기준 변경 시: 비교 문구/필터값/성장률 일괄 갱신
- 일별 날짜 변경 시:
  - 스냅샷 기준일 텍스트 갱신
  - 일별 관련 카드/퍼널/표 재계산
